Instructions to Generate SSH Keys.

STEP 1: Generate a New SSH Key

→Create a new SSH key for GitHub.

❖  Command:

→ ssh-keygen -t ed25519 -C "your_email@example.com"

❖ Explanation:

  • ssh-keygen: The command to generate SSH keys.
  • -t ed25519: Specifies the type of key. Ed25519 is modern and secure. If your system doesn't support it, use -t rsa -b 4096 for a 4096-bit RSA key.
  • -C "your_email@example.com": Adds a label (your email) to identify the key

❖ Prompt:

→ Enter file in which to save the key (/home/your_user/.ssh/id_ed25519):

❖ Action

  • If you want to overwrite the existing key, type the path of the existing key (e.g., /home/your_user/.ssh/id_ed25519) and press Enter.
  • If you're overwriting the default location, simply press Enter.

STEP 2: Confirm Overwrite

→ When prompted to overwrite the file, confirm:

❖ Prompt :

→ /home/your_user/.ssh/id_ed25519 already exists. Overwrite (y/n)?

❖ Action :

→ Type y and press Enter to overwrite the existing file.

STEP 3: Continue with SSH Key Setup

→ After overwriting, follow these steps to ensure the new key is properly configured:

  1. Start the SSH Agent:

    2. eval "$(ssh-agent -s)"

  2. Add the New Key to the Agent:

    3. ssh-add ~/.ssh/id_ed25519

  3. Copy the New Public Key:

    4. cat ~/.ssh/id_ed25519.pub

  4. Update GitHub:
    • Log in to your GitHub account.
    • Go to Settings > SSH and GPG keys > New SSH key.
    • Paste the copied public key into the "Key" field and give it a title.
    • Click Add SSH Key.

Test the Connection

❖ Command:

→ ssh -T git@github.com

❖ Expected Output:

→ Hi username! You've successfully authenticated, but GitHub does not provide shell access.

Example of a Public SSH Key

→ ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAICJ4kciWyt9SbOXkD7+7IbqZvlpF9pE1xUsU1gBRk3eB your_email@example.com

❖ Explanation:

  • Confirms that your SSH key is correctly set up and connected to your GitHub account.
